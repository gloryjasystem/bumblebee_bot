# Bumblebee Bot — Полное ТЗ Проекта

> Один умный бот — для защиты, модерации и управления всеми твоими каналами и группами в Telegram.

---

## 1. Концепция и архитектура

### Как это работает — главная идея

**Bumblebee Bot — это один бот, который подключается к нескольким твоим каналам и работает в каждом из них автоматически.**

```
@bumblebeebot
  ├── Подключён к каналу А (10k подписчиков)
  ├── Подключён к каналу Б (5k подписчиков)
  └── Подключён к группе В (2k человек)
```

Один бот — всё под контролем. Никаких дополнительных инструментов.

---

### Два режима работы одного бота

```
┌─────────────────────────────────────────────────┐
│  РЕЖИМ 1 — Личный диалог с ботом (для тебя)     │
│                                                  │

└─────────────────────────────────────────────────┘
```

---

### Путь администратора — от старта до работы

```
1. Находишь бота → /start → выбираешь язык

2. Попадаешь в главное меню (кнопки в переписке с ботом)

3. Нажимаешь «Площадки» → добавляешь свой канал/группу
   (бот должен быть администратором этого канала)

4. Настраиваешь всё через кнопки в личке:
   • Капча → включить / выбрать тип / задать текст
   • Чёрный список → загрузить файл / добавить вручную
   • Приветствие → написать свой текст с {name}
   • Защита → включить фильтры RTL, без фото, языки

5. Готово. Бот работает сам — ты больше ничего не делаешь.
```

**12.2 — Таблица всех фильтров:**

| Фильтр | Что блокирует | Сложность для пользователя |
|---|---|---|
| 🚫 Лимиты | Ограничение частоты вступлений | Средняя |
| 🌐 Фильтр по языкам | Блокировка по языку Telegram-аккаунта | Средняя |
| 🔵 RTL-символы в имени | Имена на арабском, иврите — часто боты | **Сложная** |
| 🔵 Иероглифы в имени | Имена на китайском, японском | **Сложная** |
| 🔵 Аккаунты без фото | Профили без аватарки — чаще боты/спам | **Особая** |

> ⚠️ **Техническое ограничение по фильтру «без фото»:** Telegram API не передаёт поле `has_photo` в объекте User. Для проверки требуется отдельный вызов `getUserProfilePhotos(user_id)` на каждого вступающего. Реализуется **асинхронно через `asyncio.create_task()`** после принятия заявки с обязательным `asyncio.sleep(0.1)` между запросами (защита от rate limit). Если у пользователя нет фото — кикать с задержкой ~2–5 сек после вступления.

```
(задержка < 1 секунды)
```

- Проверяет по ЧС → если есть → банит (уже после входа, задержка <1 сек)

**Когда использовать:**


## 3. База данных — PostgreSQL (Railway)

### Для чего нужна БД и как она работает

Бот хранит всё своё состояние в **PostgreSQL** на сервере Railway. Это полноценная реляционная БД, которая выдерживает миллионы записей и не теряет данные при перезагрузках.

**Что хранится в БД:**

| Что | Зачем |
|---|---|
| Владельцы бота | Тариф, язык, дата окончания тарифа |
| Площадки (каналы/группы) | Настройки: капча, фильтры, приветствие и др. |
| Чёрный список | User ID и @username заблокированных |
| Участники (для рассылки) | Все, кто вступал пока бот был подключён |
| Рассылки | История, статус, прогресс |
| Ссылки-приглашения | Статистика переходов и вступлений |
| Журнал действий | Баны, одобрения, отклонения (для тарифа Про+) |

---

### Как БД наполняется автоматически

```
Пользователь вступает в канал / проходит капчу
           ↓
Бот записывает его ID, @username, язык в таблицу bot_users
           ↓
Диалог открыт → пользователь попадает в базу рассылки
```

> База растёт сама по мере того, как приходят новые подписчики.
> Люди, которые вступили **до** подключения бота — в БД не попадают (ограничение Telegram API).

---

### Как происходит удаление из базы

Удаление из БД работает по трём сценариям:

**1. Автоматически — по событию**
```
Пользователь отписался от канала
           ↓
Бот получает ChatMemberUpdated (покинул)
           ↓
В bot_users поле left_at заполняется текущей датой
           ↓
Рассылка ему больше не отправляется
(запись сохраняется в истории статистики)
```

**2. Чёрный список — при входе или проверке**
```
Пользователь оказался в ЧС → бан
           ↓
Бот банит его в канале
           ↓
В bot_users: is_banned = true + left_at = сейчас
```

**3. Вручную — через кнопку «Найти нарушителей»**
```
Админ нажимает «Забанить всех» после проверки
           ↓
Бот банит каждого найденного пользователя
           ↓
У каждого left_at + is_banned = true
```

> Данные никогда не удаляются полностью — рассылка им просто перестаёт отправляться, а статистика (вступил / отписался) сохраняется.

## 6. Ключевые технические решения

### Управление суб-ботами
Каждый созданный пользователем бот запускается как **отдельный async polling process** или через webhook с общим диспетчером. Токен зашифрован в БД (Fernet encryption).

### Антифрод для бесплатного тарифа
- Счётчик пользователей в `bot_users` per `bot_id`
- При достижении лимита 500 — бот продолжает работать, но новых пользователей не добавляет в базу + показывает владельцу напоминание об апгрейде

### Рассылка (mass mailing)
Рассылка через очередь во избежание бана:
- Максимум 25 сообщений/секунду (лимит Telegram API)
- При ошибке "Too Many Requests" — пауза и retry
- Прогресс отображается в реальном времени (обновление сообщения)

## 8. База данных — Полная схема PostgreSQL (Railway)

### Почему PostgreSQL на Railway, а не SQLite?

| Критерий | SQLite | PostgreSQL on Railway |
|---|---|---|
| Конкурентные запросы | ❌ Блокировки при нагрузке | ✅ До 10,000 соединений |
| Объём данных | До ~1GB надёжно | ТБ без ограничений |
| Репликация | ❌ | ✅ Автоматические бэкапы |
| Шифрование | ❌ | ✅ TLS + at-rest encryption |
| Масштабирование | ❌ | ✅ Railway авто-скейлинг |
| Индексы для ЧС 100k+ | Медленно | ✅ Быстро (B-tree / GIN) |

### Полная схема таблиц

```sql
-- 1. Пользователи платформы (владельцы бота)
CREATE TABLE platform_users (
    user_id         BIGINT PRIMARY KEY,             -- Telegram user ID
    username        VARCHAR(64),
    language        CHAR(2) DEFAULT 'ru',
    tariff          VARCHAR(16) DEFAULT 'free',     -- free | start | pro | business
    tariff_until    TIMESTAMPTZ,
    trial_used      BOOLEAN DEFAULT false,
    created_at      TIMESTAMPTZ DEFAULT now()
);

-- 2. Боты (зарезервировано — для будущей логики подключения своего бота)
CREATE TABLE bots (
    id              SERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    token_encrypted BYTEA NOT NULL,                 -- AES-256 зашифрованный токен
    username        VARCHAR(64),
    display_name    VARCHAR(128),
    status          VARCHAR(16) DEFAULT 'active',   -- active | paused | deleted
    settings        JSONB DEFAULT '{}',
    created_at      TIMESTAMPTZ DEFAULT now()
);

-- 3. Площадки (каналы/группы)
CREATE TABLE bot_chats (
    id              SERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    chat_id         BIGINT NOT NULL,
    chat_username   VARCHAR(64),
    chat_title      VARCHAR(256),
    chat_type       VARCHAR(16),                    -- channel | supergroup | group
    is_premium      BOOLEAN DEFAULT false,
    is_active       BOOLEAN DEFAULT true,
    settings        JSONB DEFAULT '{}',             -- все настройки площадки (капча, фильтры и др.)
    added_at        TIMESTAMPTZ DEFAULT now(),
    UNIQUE(owner_id, chat_id)
);

-- 4. Чёрный список (оптимизирован для 500k+ записей)
CREATE TABLE blacklist (
    id              BIGSERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    user_id         BIGINT,                         -- NULL если только username
    username        VARCHAR(64),
    reason          VARCHAR(256),
    added_by        BIGINT,                         -- Telegram ID того, кто добавил
    added_at        TIMESTAMPTZ DEFAULT now()
);
-- Индексы для мгновенного поиска:
CREATE INDEX idx_blacklist_owner_userid   ON blacklist(owner_id, user_id) WHERE user_id IS NOT NULL;
CREATE INDEX idx_blacklist_owner_username ON blacklist(owner_id, username) WHERE username IS NOT NULL;

-- 5. Участники (для рассылки и проверки нарушителей)
CREATE TABLE bot_users (
    id              BIGSERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    chat_id         BIGINT NOT NULL,               -- реальный Telegram chat_id
    user_id         BIGINT NOT NULL,
    username        VARCHAR(64),
    first_name      VARCHAR(128),
    language_code   CHAR(5),
    has_photo       BOOLEAN,
    is_premium      BOOLEAN DEFAULT false,
    has_rtl         BOOLEAN DEFAULT false,          -- RTL символы в имени
    has_hieroglyph  BOOLEAN DEFAULT false,
    is_active       BOOLEAN DEFAULT true,           -- не отписался
    bot_activated   BOOLEAN DEFAULT false,          -- написал боту (канал рассылки открыт)
    joined_at       TIMESTAMPTZ DEFAULT now(),
    last_seen       TIMESTAMPTZ,
    UNIQUE(owner_id, chat_id, user_id)
);

-- 6. Ссылки-приглашения
CREATE TABLE invite_links (
    id              SERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    chat_id         BIGINT NOT NULL,
    name            VARCHAR(256),
    link            VARCHAR(256) UNIQUE,
    link_type       VARCHAR(16),                    -- regular | request | once
    budget          DECIMAL(10,2),
    currency        CHAR(3),
    usage_limit     INTEGER,
    uses_count      INTEGER DEFAULT 0,
    unsubscribed    INTEGER DEFAULT 0,
    auto_accept     VARCHAR(16) DEFAULT 'basic',    -- basic | on | off
    created_at      TIMESTAMPTZ DEFAULT now()
);

-- 7. Рассылки
CREATE TABLE mailings (
    id              SERIAL PRIMARY KEY,
    owner_id        BIGINT REFERENCES platform_users(user_id) ON DELETE CASCADE,
    chat_id         BIGINT NOT NULL,
    content         JSONB NOT NULL,                 -- текст + медиа + кнопки
    status          VARCHAR(16) DEFAULT 'draft',    -- draft | scheduled | running | paused | done | cancelled
    speed           VARCHAR(8) DEFAULT 'low',       -- low | medium | high
    scheduled_at    TIMESTAMPTZ,
    started_at      TIMESTAMPTZ,
    finished_at     TIMESTAMPTZ,
    total           INTEGER DEFAULT 0,
```

### 9.1 — Валидация Telegram WebApp (tg.initData) ⚠️ КРИТИЧНО

**Проблема:** Без проверки `initData` любой может вызвать `/api/create-crypto-payment` с чужим `user_id` и создать платёж от чужого имени.

**Решение:** Верифицировать HMAC-SHA256 подпись Telegram перед любым действием с WebApp.

```python
import hashlib, hmac, time, json
from urllib.parse import unquote, parse_qsl

def verify_init_data(init_data: str, bot_token: str) -> dict | None:
    """
    Возвращает dict с данными пользователя если OK,
    None — если подпись неверна или данные устарели (> 1 часа).
    """
    params = dict(parse_qsl(unquote(init_data), keep_blank_values=True))
    received_hash = params.pop("hash", None)
    if not received_hash:
        return None

    data_check = "\n".join(f"{k}={v}" for k, v in sorted(params.items()))
    secret = hmac.new(b"WebAppData", bot_token.encode(), hashlib.sha256).digest()
    expected = hmac.new(secret, data_check.encode(), hashlib.sha256).hexdigest()

    if not hmac.compare_digest(expected, received_hash):
        return None
    if abs(time.time() - int(params.get("auth_date", 0))) > 3600:
        return None  # Данные старее 1 часа

    return json.loads(params.get("user", "{}"))

# Использование в API endpoint:
@app.post("/api/create-crypto-payment")
async def create_crypto_payment(request: Request):
    data = await request.json()
    user = verify_init_data(data.get("init_data", ""), BOT_TOKEN)
    if not user:
        return JSONResponse({"error": "Unauthorized"}, status_code=401)
    user_id = user["id"]  # Берём из ВЕРИФИЦИРОВАННЫХ данных, не из запроса
    # ...
```

> Фронтенд передаёт `window.Telegram.WebApp.initData` в каждом запросе к бэкенду.

### 9.2 — Верификация NOWPayments Webhook (HMAC-SHA512) ⚠️ КРИТИЧНО

**Проблема:** Без проверки подписи кто угодно может отправить фейковый POST на `/nowpayments/webhook` и бесплатно активировать тариф.

**Решение:** NOWPayments подписывает каждый webhook HMAC-SHA512. Проверяем ОБЯЗАТЕЛЬНО.

```python
def verify_nowpayments_sig(payload_bytes: bytes, signature: str, ipn_secret: str) -> bool:
    """
    payload_bytes — сырые байты тела запроса (ДО декодирования json!)
    """
    data = json.loads(payload_bytes)
    sorted_str = json.dumps(data, sort_keys=True, separators=(",", ":"))
    expected = hmac.new(ipn_secret.encode(), sorted_str.encode(), hashlib.sha512).hexdigest()
    return hmac.compare_digest(expected, signature)

@app.post("/nowpayments/webhook")
async def nowpayments_webhook(request: Request):
    payload = await request.body()                    # Сырые байты — ВАЖНО
    sig = request.headers.get("x-nowpayments-sig", "")
    if not verify_nowpayments_sig(payload, sig, NOWPAYMENTS_IPN_SECRET):
        return JSONResponse({"status": "forbidden"}, status_code=403)
    # ...
```

## 10. Чёрный список — Архитектура, логика и полный функционал

> Это **центральная функция** Bumblebee Bot. Именно ЧС позволяет автоматически защищать каналы от ботов, спамеров и нежелательных пользователей — без ручной работы.

---

### 10.1 — Архитектура базы ЧС

**ЧС привязан к владельцу (`owner_id`), а НЕ к отдельному каналу.**
Один и тот же список автоматически защищает **все подключённые площадки** одновременно.

```
Владелец (owner_id: 123456789)
    ├── Канал А  ─┐
    ├── Канал Б  ─┼── Один ЧС → защищает все три площадки
    └── Группа В ─┘
```

**Хранение:** таблица `blacklist` в PostgreSQL.
**Поиск:** составной индекс `(owner_id, user_id)` и `(owner_id, username)` — мгновенный при любом размере базы (проверено до 1,000,000 записей).

**Лимиты по тарифу:**

| Тариф | Лимит ЧС |
|---|---|
| 🆓 Free | Хранение ЧС недоступно. Авто-бан работает, но база не сохраняется между сессиями |
| 🌱 Старт | до 10,000 записей |
| ⭐ Про | до 100,000 записей |
| 💼 Бизнес | до 1,000,000 записей |

---

### 10.2 — Три способа добавить пользователя в ЧС

| Способ | Формат | Назначение |
|---|---|---|
| Вручную — один | `@username` или `123456789` | Быстрый точечный бан |
| Вручную — несколько | Несколько через пробел или с новой строки | Небольшие списки |
| Файлом TXT/CSV | Файл до 20 MB, до 100k строк | Массовые базы ботов и спамеров |

**Форматы файлов — поддерживаются все популярные:**

```
TXT — каждая запись с новой строки:
  123456789
  @spammer_nick
  987654321

CSV — колонка user_id или username (заголовок необязателен):
  123456789
  @username1
  @username2
```

**Обработка при импорте:**
- Строки с неверным форматом → пропускаются (считаются, выводятся в отчёт)
- Дублирующиеся записи → `ON CONFLICT DO NOTHING` (без ошибок)
- @username без user_id → сохраняется как username (user_id будет заполнен при первом вступлении)

### 10.3 — Авто-бан при вступлении (реальное время)

**Сценарий 1 — Закрытый канал (с заявками):**
Пользователь из ЧС **не попадает в канал вообще** — заявка отклоняется до входа.

```
Человек нажимает «Подать заявку»
           ↓
Telegram API → ChatJoinRequest event → боту
           ↓
Бот проверяет ЧС:
SELECT 1 FROM blacklist
  WHERE owner_id = $1
    AND (user_id = $2 OR username = $3)
  LIMIT 1
           ↓
Нашлось           →  declineChatJoinRequest() + banChatMember()
                       Человек НЕ видит содержимого канала вообще
Не нашлось        →  Обычная обработка (капча / авто-одобрение)
```

**Сценарий 2 — Открытый канал (без заявок):**
Технически пользователь подписывается, но бот банит его в течение < 1 секунды.

```
Человек нажал «Подписаться»
           ↓
Telegram API → ChatMemberUpdated (status: member) → боту
           ↓
Бот проверяет ЧС (тот же SQL-запрос)
           ↓
Нашлось           →  banChatMember() (задержка < 1 сек)
                       + в bot_users: is_active = false
Не нашлось        →  Приветствие + запись в bot_users
```

> ⚠️ **Ограничение Telegram API:** в открытом канале полная блокировка **до** входа технически невозможна — человек успевает подписаться на долю секунды. Для блокировки **до входа** нужен закрытый канал с заявками.

### 10.4 — Авто-зачистка при добавлении в ЧС

**Ключевая функция.** Как только владелец добавляет пользователя в ЧС — бот **немедленно** проверяет все подключённые активные площадки и банит его там, если он уже является участником.

**Флоу при добавлении одного пользователя вручную:**

```
Владелец добавляет @baduser в ЧС
           ↓
Бот сохраняет запись: INSERT INTO blacklist (owner_id, user_id, username, ...)
           ↓
Бот обходит все активные площадки владельца:
  SELECT chat_id FROM bot_chats
    WHERE owner_id = ? AND is_active = true
           ↓
Для каждой площадки → getChatMember(chat_id, user_id)
           ↓
Если участник → banChatMember(chat_id, user_id)
              + UPDATE bot_users SET is_active=false WHERE ...
           ↓
Итоговое уведомление владельцу:
«✅ @baduser добавлен в ЧС.
 🚫 Забанен в 2 площадках: Crypto Alpha, Trading Hub
 ⚪ Не найден в: Группа В»
```

**Флоу при загрузке файла (массовая авто-зачистка):**

```
Загружен файл → 8,430 записей импортировано
           ↓
Бот запускает фоновую задачу sweep_after_import()
           ↓
Для каждой активной площадки:
  SELECT bu.user_id FROM bot_users bu
  INNER JOIN blacklist bl ON bl.owner_id = bu.owner_id
    AND (bl.user_id = bu.user_id OR bl.username = bu.username)
  WHERE bu.owner_id = ? AND bu.chat_id = ? AND bu.is_active = true
           ↓
Пакетный бан → banChatMember() для каждого нарушителя
           ↓
Уведомление:
«✅ Зачистка завершена.
 🚫 Забанено: 47 человек
 Площадки: Crypto Alpha (31), Trading Hub (16)»
```

> Зачистка работает **только с теми, кто вступил пока бот был подключён** (ограничение Telegram API — полный список участников получить невозможно).

```python
    """
    chats = await db.fetch(
        "SELECT chat_id FROM bot_chats WHERE owner_id = $1 AND is_active = true",
        owner_id
    )
    total_banned = 0

    for chat in chats:
        violators = await db.fetch("""
            SELECT DISTINCT bu.user_id
            FROM bot_users bu
            INNER JOIN blacklist bl
              ON bl.owner_id = bu.owner_id
              AND (
                (bl.user_id IS NOT NULL AND bl.user_id = bu.user_id)
                OR
                (bl.username IS NOT NULL AND bl.username = bu.username)
              )
            WHERE bu.owner_id = $1
              AND bu.chat_id = $2
              AND bu.is_active = true
        """, owner_id, chat["chat_id"])

        for v in violators:
            try:
                await bot.ban_chat_member(chat["chat_id"], v["user_id"])
                await db.execute(
                    """UPDATE bot_users SET is_active = false
                       WHERE owner_id = $1 AND chat_id = $2 AND user_id = $3""",
                    owner_id, chat["chat_id"], v["user_id"]
                )
                total_banned += 1
            except Exception:
                pass  # Уже забанен или покинул канал — игнорируем

    return total_banned
```

| | Ручная проверка | NOWPayments |

```sql
CREATE TABLE payments (
    id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id     BIGINT REFERENCES platform_users(user_id),
    tariff      VARCHAR(16),          -- start | pro | business
    period      VARCHAR(8),           -- month | year

    created_at  TIMESTAMPTZ DEFAULT now(),

);
CREATE INDEX idx_payments_user ON payments(user_id, status);
CREATE INDEX idx_payments_np   ON payments(np_payment_id) WHERE np_payment_id IS NOT NULL;
```

```python
async def create_crypto_payment(request: Request):
    data = await request.json()
    tariff   = data["tariff"]    # "pro"
    period   = data["period"]    # "month"

    # Вызываем NOWPayments API
```

```python
@app.post("/nowpayments/webhook")
async def nowpayments_webhook(request: Request):
    data      = await request.json()
    signature = request.headers.get("x-nowpayments-sig")

    # Проверяем подпись (HMAC-SHA512)
    if not verify_nowpayments_signature(data, signature, NOWPAYMENTS_IPN_SECRET):
        return {"status": "invalid signature"}

    if data["payment_status"] == "finished":
        payment_id = data["order_id"]   # UUID из нашей БД
        row = await db.fetchrow(
            "SELECT user_id, tariff, period FROM payments WHERE id=$1",
            payment_id
        )
        if row:
            await db.execute(
                "UPDATE payments SET status='paid', paid_at=now() WHERE id=$1",
                payment_id
            )
            await activate_tariff(row["user_id"], row["tariff"], row["period"])
            await notify_user_paid(row["user_id"], row["tariff"], row["period"])
            await notify_owner_payment(row["user_id"], data)

    return {"status": "ok"}
```
